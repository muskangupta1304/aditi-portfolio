CalcQuest:
Learning Through the City

// Game Design · Integer Operations · Educational Game Design Document

"What if every wrong move in maths was just the beginning of a smarter strategy?"

DESIGN PREMISE
01 — Overview

A City Where Every Route Is a Calculation

CalcQuest is a 3D city navigation game built in Unity that transforms integer operations from an abstract classroom exercise into something a player feels. The core insight: when mathematical operations become physical choices in a world you're moving through, they stop being symbols on a page.

Players navigate a city map on a bicycle, choosing routes that apply mathematical operations — addition or subtraction — in pursuit of a target score. Every decision has a consequence. Every route is a calculation. The game doesn't explain integers — it makes you use them.

Target Audience: Grades 6–8 · Ages 11–14

03 — Core Mechanic

The Color-Coded City

The game's central design decision: color as mathematical scaffolding. Instead of reading a sign that says "add" or "subtract," players navigate roads that are visually coded — making the operations feel spatial and intuitive, not symbolic.

🟩 Green Paths

Positive routes — riding a green path adds to your score. Players learn to seek these out when their score is below target.

🟥 Red Paths

Negative routes — riding a red path subtracts from your score. Avoiding or strategically using red paths builds integer number sense.

05 — Design Principles

How Every Mechanic Serves a Learning Goal

🗺️

Spatial Navigation

Moving through a map makes mathematical choices feel physical. Route decisions become number decisions — learners aren't calculating, they're navigating.

⏱️

Time Constraint

The countdown timer creates urgency without overwhelming. It encourages quick mental estimation — the skill that underpins all integer fluency.

📊

Real-Time Score Feedback

The score updates the moment a route is taken. Immediate feedback reinforces cause and effect — learners see the consequence of each decision instantly.

🎯

Target Goal Design

Reaching exactly the target score — not just getting close — requires strategic route planning. This deepens the mathematical thinking beyond simple addition and subtraction.

What I Learned

Designing for learning is fundamentally different from designing for entertainment. Every mechanic in CalcQuest had to carry conceptual weight — the color coding wasn't decoration, the timer wasn't arbitrary pressure. Each design decision was a pedagogical decision. Understanding this distinction — that the game's feel had to match the learning intent — changed how I think about educational game design entirely.
